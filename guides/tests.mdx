A **test** is a reusable instrument you save under **Library → Tests**. Load it onto a [simulation](/guides/simulations) as a **Test** step to measure how a cohort reacts to creative or messaging — with variants and segmented metrics.

  <img src="/images/app/tests-list.png" alt="Tests library listing Cost-of-Living, Healthcare, and Tax Relief instruments" />
</Frame>

## Create a library test

Go to **Library → Tests → New test**. Configure the design fields that travel with the instrument:

### Identity

Name the test and state a **hypothesis** — what you expect to happen and why.

### Creative

Choose the format:

- **Video** — react to a video cut.
- **Image** — react to a static asset.
- **Messaging** — react to copy you type or upload.

Add **variants** to compare options side by side — up to four, labelled A through D. Attach an asset or message to each when you place the test on a canvas. See [Assets](/guides/creatives).

### Metrics

Select the quantitative measures you want scored, such as **Recall**, **Sentiment**, and **Purchase intent**. Each is reported out of 100 and can be cut by segment.

### Questions

Add the questions each persona answers — open-ended, rating, or multiple choice. A good test mixes a few open-ended prompts with structured ratings.

## Use a test in a simulation

On a simulation's **Model** tab, click **Add node → Test**. Load a library test or configure from scratch. Wire:

- A **Cohort** node for respondents
- An **Asset** node for each variant (optional for messaging)
- Optional **Knowledge** / **News** for context

<Frame caption="The Model canvas — a test step wired to a cohort, with a focus group step downstream.">
  <img src="/images/app/test-builder.png" alt="Simulation model canvas with cohort, test, and focus group nodes connected" />
</Frame>

Audience, world state, and run-time overlays stay on the canvas step. Design fields resolve from the library instrument at run time when linked.

## Read results

Open the simulation's **Results** tab, select a run, and open the test step:

- Headline metrics (recall, sentiment, intent, clarity)
- Segment cuts and lift vs control
